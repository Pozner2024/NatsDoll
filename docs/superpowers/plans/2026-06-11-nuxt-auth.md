# Auth на клиенте (спек 4) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Авторизация в Nuxt работает как в старом SPA (логин, silent refresh, refresh-on-401, логаут, OAuth callback, verify-email, reset-password); старые `src/router` и `src/App.vue` удалены.

**Architecture:** Client-плагин восстанавливает связку `setupAuthInterceptor` + `initAuth` из удалённого `src/main.ts`. Named middleware `auth`/`admin` — заготовки логики старого `beforeEach` для спеков 5–6. Три auth-страницы — тонкие обёртки с `routeRules: { ssr: false }`. Кастомный scrollBehavior переезжает в `app/router.options.ts`. Спек: `docs/superpowers/specs/2026-06-11-04-nuxt-auth-design.md`.

**Tech Stack:** Nuxt 4 (plugins, route middleware, routeRules, router.options), Pinia, Playwright MCP (живые сценарии).

**Контекст для исполнителя без знания проекта:**
- Монорепо npm workspaces, фронтенд `apps/web` (Nuxt 4, ФСД в `src/`, алиас `@` → `src`). Ветка `feat/nuxt-skeleton`. Dev — Docker Compose (web:5173, api:3000).
- **Nuxt в Docker на Windows не видит НОВЫЕ файлы через bind mount** — после создания файлов в `apps/web` и правок `nuxt.config.ts` нужен `docker compose restart web`, затем дождаться 200 от `http://localhost:5173/`.
- НЕ добавлять комментарии в код. Сообщения коммитов заканчивать строкой: `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`
- Тесты web: `node --max-old-space-size=4096 ./node_modules/vitest/vitest.mjs run --root apps/web --reporter=basic` (без `--root` — ложное падение всех .vue).
- `'nuxt/app'` экспортирует `defineNuxtPlugin`, `defineNuxtRouteMiddleware`, `navigateTo` — импортировать явно (проектный паттерн).
- Креды админа для живой проверки: `ADMIN_EMAIL`/`ADMIN_PASSWORD` в `apps/api/.env` (строки 20–21).

```ts
    getAccessToken: () => authStore.accessToken,
```

### Task 2: Named middleware auth и admin

**Files:**
- Create: `apps/web/app/middleware/auth.ts`
- Create: `apps/web/app/middleware/admin.ts`

- [ ] **Step 1: Создать `apps/web/app/middleware/auth.ts`**

```ts
import { defineNuxtRouteMiddleware, navigateTo } from 'nuxt/app'
import { useAuthStore } from '@/entities/user'
import { useAuthModal } from '@/shared'

export default defineNuxtRouteMiddleware(async (to) => {
  if (import.meta.server) return

  const authStore = useAuthStore()
  if (!authStore.authReady) await authStore.initAuth()

  if (!authStore.isLoggedIn) {
    const { open } = useAuthModal()
    open()
    return navigateTo('/')
  }

  if (to.path.startsWith('/account') && authStore.user?.role === 'ADMIN') {
    return navigateTo('/admin')
  }
})
```

- [ ] **Step 2: Создать `apps/web/app/middleware/admin.ts`**

```ts
import { defineNuxtRouteMiddleware, navigateTo } from 'nuxt/app'
import { useAuthStore } from '@/entities/user'

export default defineNuxtRouteMiddleware(async () => {
  if (import.meta.server) return

  const authStore = useAuthStore()
  if (!authStore.authReady) await authStore.initAuth()

  if (!authStore.isLoggedIn || authStore.user?.role !== 'ADMIN') {
    return navigateTo('/')
  }
})
```

```ts
import type { RouterConfig } from '@nuxt/schema'

export default {
  scrollBehavior(to, _from, savedPosition) {
    if (savedPosition) return savedPosition
    if (to.hash) return { el: to.hash, behavior: 'smooth' }
    return { top: 0 }
  },
} satisfies RouterConfig
```
