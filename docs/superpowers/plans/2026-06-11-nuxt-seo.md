# SEO-обвес (спек 3) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Каждая публичная страница отдаёт title/description/OG/canonical в server-HTML, товар — JSON-LD `Product`; сайт отдаёт robots.txt и динамический sitemap.xml.

**Architecture:** Меты статичных страниц — в обёртках `app/pages/*` через `useSeoMeta`/`useHead` (импорт из `'nuxt/app'`); меты и JSON-LD товара — в `ProductPageWidget.vue` (данные уже там). `sitemap.xml` — Nitro server-route, тянет данные из API: существующий `/categories` и новый лёгкий `/products/sitemap-data`. `robots.txt` — статичный файл. Спек: `docs/superpowers/specs/2026-06-11-03-nuxt-seo-design.md`.

**Tech Stack:** Nuxt 4 (useSeoMeta, useHead, runtimeConfig, Nitro server routes), Hono + Prisma (новый эндпоинт), Vitest.

**Контекст для исполнителя без знания проекта:**
- Монорепо npm workspaces: фронтенд `apps/web` (Nuxt 4, ФСД в `src/`, алиас `@` → `src`), бэкенд `apps/api` (Hono, 3 слоя: application/infrastructure/presentation, composition root — `app.ts`). Ветка `feat/nuxt-skeleton`.
- Dev-окружение — Docker Compose (web:5173, api:3000, db:5432). **Nuxt в Docker на Windows не видит НОВЫЕ файлы через bind mount** — после создания нового файла в `apps/web` нужен `docker compose restart web` (правки существующих подхватываются HMR). Изменение `nuxt.config.ts` тоже требует рестарта.
- НЕ добавлять комментарии в код. НЕ трогать файлы вне списка задачи. Сообщение коммита заканчивать строкой: `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`
- Тесты web: `node --max-old-space-size=4096 ./node_modules/vitest/vitest.mjs run --root apps/web --reporter=basic` (без `--root` — ложное падение всех .vue).
- Тесты api: `node --max-old-space-size=4096 ./node_modules/vitest/vitest.mjs run --root apps/api --reporter=basic` (без `--root` — не грузится vitest.setup.ts с секретами).

export const DEFAULT_OG_IMAGE = 'https://storage.yandexcloud.net/natsdoll/gallery1/p1.webp'

const META_DESCRIPTION_MAX = 160

export function metaDescription(source: string): string {
  const text = source.replace(/<[^>]*>/g, ' ').replace(/\s+/g, ' ').trim()
  if (text.length <= META_DESCRIPTION_MAX) return text
  const cut = text.slice(0, META_DESCRIPTION_MAX - 1)
  const lastSpace = cut.lastIndexOf(' ')
  return `${cut.slice(0, lastSpace > 0 ? lastSpace : cut.length)}…`
}

- [ ] **Step 4: Заменить `apps/web/app/pages/shop/[[category]].vue`**

Title/description зависят от категории; имя категории — из `categoryStore`, который SSR-гидратируется layout'ом (спек 2). Реактивные значения — через getter-функции (`useSeoMeta` их поддерживает); `useHead` с функцией-аргументом — для реактивного canonical при клиентской навигации между категориями:

```vue
<template>
  <ShopPage />
</template>

<script setup lang="ts">
import { computed } from 'vue'
import { useRoute } from 'vue-router'
import { useSeoMeta, useHead, useRuntimeConfig } from 'nuxt/app'
import ShopPage from '@/pages/ShopPage.vue'
import { useCategoryStore } from '@/entities/category'
import { DEFAULT_OG_IMAGE } from '@/shared'

definePageMeta({ name: 'shop' })

const siteUrl = useRuntimeConfig().public.siteUrl
const route = useRoute()
const categoryStore = useCategoryStore()

const categoryName = computed(() => {
  const slug = route.params.category
  if (typeof slug !== 'string' || slug.length === 0) return null
  return categoryStore.categories.find((c) => c.slug === slug)?.name ?? null
})
const title = computed(() =>
  categoryName.value ? `${categoryName.value} — NatsDoll` : 'The Shop — NatsDoll',
)
const description = computed(() =>
  categoryName.value
    ? `Handmade polymer clay ${categoryName.value.toLowerCase()} by NatsDoll. Every piece is sculpted and painted by hand.`
    : 'Browse handmade polymer clay dolls, birthday gifts, Christmas ornaments and personalized keepsakes. Worldwide shipping.',
)
const canonical = computed(() => `${siteUrl}${route.path}`)

useSeoMeta({
  title,
  description,
  ogTitle: title,
  ogDescription: description,
  ogImage: DEFAULT_OG_IMAGE,
  ogUrl: canonical,
  ogType: 'website',
  twitterCard: 'summary_large_image',
})
useHead(() => ({ link: [{ rel: 'canonical', href: canonical.value }] }))
</script>
```

### Task 9: sitemap.xml — Nitro server-route

**Files:**
- Create: `apps/web/server/routes/sitemap.xml.ts`

- [ ] **Step 1: Создать route**

`apps/web/server/routes/sitemap.xml.ts` (`defineEventHandler`, `setHeader`, `useRuntimeConfig`, `$fetch` — глобальные авто-импорты Nitro; slug'и URL-safe, XML-экранирование не требуется):

```ts
type SitemapProduct = { slug: string; updatedAt: string }
type SitemapCategory = { slug: string }

function urlTag(loc: string, lastmod?: string): string {
  return `  <url><loc>${loc}</loc>${lastmod ? `<lastmod>${lastmod}</lastmod>` : ''}</url>`
}

export default defineEventHandler(async (event) => {
  const apiBase = process.env.NUXT_API_INTERNAL_URL ?? 'http://localhost:3000'
  const siteUrl = useRuntimeConfig(event).public.siteUrl

  const [products, categories] = await Promise.all([
    $fetch<SitemapProduct[]>(`${apiBase}/products/sitemap-data`),
    $fetch<SitemapCategory[]>(`${apiBase}/categories`),
  ])

  const urls = [
    urlTag(`${siteUrl}/`),
    urlTag(`${siteUrl}/gallery`),
    urlTag(`${siteUrl}/shop`),
    ...categories.map((c) => urlTag(`${siteUrl}/shop/${c.slug}`)),
    ...products.map((p) => urlTag(`${siteUrl}/product/${p.slug}`, p.updatedAt.slice(0, 10))),
  ]

  setHeader(event, 'content-type', 'application/xml')
  return `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
${urls.join('\n')}
</urlset>`
})
```

Ошибка API внутри `$fetch` пробрасывается — Nitro ответит 500 (по спеку: честная ошибка лучше пустого sitemap).
