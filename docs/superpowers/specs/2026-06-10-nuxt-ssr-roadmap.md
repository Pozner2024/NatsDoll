# Roadmap: миграция на Nuxt SSR ради видимости в Google

Дата: 2026-06-10. Статус: утверждён пользователем.

## Цель

Сделать natsdoll.com видимым для Google. Сейчас сайт — Vue 3 SPA без SSR,
без robots.txt, без sitemap.xml, с единственным статичным `<title>NatsDoll</title>`,
без OG-тегов и структурированных данных. Решение — миграция на Nuxt 4 с SSR
для публичных страниц и полным SEO-обвесом.

## Принятые решения

- **Полный SSR через Nuxt 4** (а не quick wins на SPA, не пререндеринг, не Vike).
- **SSR только для публичных страниц**: главная, магазин, товар, галерея.
  Кабинет, корзина, админка, auth-страницы — client-only (`routeRules: { ssr: false }`).
  JWT-авторизация на сервер не переносится, security-инварианты не меняются.
- **Без кэширования HTML на старте.** SSR рендерит каждый запрос свежим — изменения
  в БД (заголовки, описания, цены товаров) видны сразу, без пересборки и деплоя.
  Если трафик вырастет — добавить `swr` в `routeRules` с TTL 60–300 с.
  Инвариант: кэшировать можно только страницы без пользовательских данных.
- **Прод живёт на текущем SPA** до завершения спека 7. Работа идёт в отдельной ветке.

## Спеки

Каждый спек — отдельный цикл brainstorm → spec → plan → реализация.
Спеки 4–6 независимы друг от друга (после 1–2), спек 7 — последний.

| # | Спек | Результат / критерий приёмки |
|---|------|------------------------------|
| 1 | Nuxt-каркас | Nuxt 4 установлен, ФСД-слои и алиасы сохранены, SCSS/шрифты/CSS-переменные работают, минимальная главная собирается и рендерится на сервере. Существующие unit-тесты проходят. |
| 2 | Публичные страницы с SSR | Главная, магазин, товар, галерея рендерятся на сервере с данными из API (`useFetch` + `runtimeConfig.apiBase`: сервер ходит на `api:3000`, браузер на `/api`). Шапка рендерится анонимно, auth-состояние догидрачивается на клиенте. |
| 3 | SEO-обвес | `useSeoMeta` на каждой публичной странице (title, description, OG, canonical), `robots.txt` с указанием sitemap, динамический `sitemap.xml` (server-route Nitro, товары из API, `lastmod`), JSON-LD `Product` (цена, валюта, наличие, фото). Всё видно в curl-ответе без выполнения JS. |
| 4 | Auth на клиенте | Pinia-сторы, auth-модалка, route-guards → Nuxt middleware, страницы auth-callback / verify-email / reset-password (client-only). Логин, логаут, refresh-rotation работают как раньше. |
| 5 | Кабинет + корзина | Все страницы `/account/**` и `/cart` работают client-only. |
| 6 | Админка | `/admin/**` работает client-only, включая загрузку изображений. |
| 7 | Деплой | Dockerfile с Nitro-сервером вместо статики, docker-compose, Caddy проксирует на web-контейнер вместо `file_server`, правки CSP, CI-pipeline. Sitemap отправлен в Google Search Console. |

## Известные риски

- **Гидратационные mismatch'и** — всё, что зависит от `localStorage`/auth при первом
  рендере, оборачивать в `<ClientOnly>`.
- **Кириллица в пути + tsc OOM** — проверить, что обходные команды из CLAUDE.md
  (root-tsc с увеличенным heap) переживают добавление Nuxt-слоя typecheck.
- **Vitest** — unit-тесты `.vue` сейчас требуют `--root apps/web`; после миграции
  проверить совместимость с тулчейном Nuxt.
